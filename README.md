## 🧩 What diffsense-api Does

diffsense-api takes changed code and turns it into plain text that is easier to read.

You can use it to create:

- Commit messages for Git history
- Security review notes for changed files
- Changelog entries for releases
- Short summaries for code reviews
- Clean text from raw git diffs

This helps when you want to explain what changed without writing the text by hand.

## 🛠️ How to Use It

1. Open your Git tool, editor, or terminal.
2. Copy the diff you want to review.
3. Send the diff to diffsense-api.
4. Choose the output type you need.
5. Read the generated text.
6. Copy it into your commit, review, or changelog.

Example use cases:

- You finish a bug fix and need a commit message
- You check a pull request and want a security review note
- You prepare a release and need a changelog item
- You want a short summary for a large code change

## 🔍 Example Results

## 🧱 Basic Troubleshooting

### The app does not open

- Make sure you downloaded the right file
- Try running it again as administrator
- Check if Windows blocked the file
- Re-extract the ZIP file if it looks incomplete

### The browser does not show the app

- Wait a few seconds after launch
- Check the local address in the app window
- Make sure the app is still running
- Try another browser

### The API does not respond

- Confirm the app is open
- Check the port or address shown in the window
- Restart the app
- Make sure another app is not using the same port

### Windows blocks the file

- Right-click the file
- Choose Properties
- Look for an Unblock option
- Apply the change
- Try opening the file again
